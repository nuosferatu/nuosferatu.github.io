OpenGL - 帧缓冲

这个系列是我学习 OpenGL 过程中记录的笔记，本节主要是帧缓冲及其应用，教程原文看这里。

帧缓存（framebuffer）包含：

  • 颜色缓冲
  • 深度缓冲
  • 模板缓冲

这也是我们默认使用的默认帧缓冲，由 GLFW 自动创建。

一个完整的帧缓冲，需要满足下列条件：

  • 附加至少一个缓冲（颜色、深度或模板缓冲）
  • 至少有一个颜色附件（attachment）
  • 所有的附件都必须是完整的（保留了内存）
  • 每个缓冲都应该有相同的样本数

附加附件

完整性检查前，需要给帧缓冲附加附件，附件是一个内存位置，可以作为帧缓冲的一个缓冲，类似一个图像。创建一个附件有两个选项：

  • 纹理附件
  • 渲染缓冲对象（renderbuffer object）附件

纹理附件

当把一个纹理附加到帧缓冲的时候，所有的渲染指令将会写入到这个纹理中，就想它是一个普通的颜色、深度或模板缓冲一样。使用纹理的优点是，所有渲染操作的结果将会被储存在一个纹理图像中，我们之后可以在着色器中很方便地使用它。

参数：

  • target，帧缓冲的目标，GL_FRAMEBUFFER、GL_READ_FRAMEBUFFER 或 GL_DRAW_FRAMEBUFFER
  • attachment，附件类型，GL_COLOR_ATTACHMENT0 是一个颜色附件，0 是编号，表示可以添加多个
  • textarget，纹理类型
  • texture，要附加的纹理本身
  • level，多级渐远纹理的级别，保留为 0

附件类型还可以是深度和模板缓冲，要添加深度缓冲，附件类型为 GL_DEPTH_ATTACHMENT，同时纹理的格式（format）和内部格式（internal format）类型变为 GL_DEPTH_COMPONENT，来反映深度缓冲的储存格式。

如果附件模板缓存，则是 GL_STENCIL_ATTACHMENT 和 GL_STENCIL_INDEX。

渲染缓存对象附件

和纹理图像一样，渲染缓冲对象是一个真正的缓冲，即一系列的字节、整数、像素等。渲染缓冲对象附加的好处是，它会将数据储存为 OpenGL 原生的渲染格式，它是为离屏渲染到帧缓冲优化过的。

渲染缓冲对象是只写的，不可单独读取。

渲染缓冲对象直接把所有渲染数据存储在缓冲中，不做任何纹理格式转换，数据原生，因此复制到其他缓冲会很快，是快速可写储存介质，每个渲染迭代最后使用的 glfwSwapBuffers 也可以用渲染缓冲对象实现：写入一个渲染缓冲对象，最后交换到另一个渲染缓冲就可以了，完美！

如何选择纹理附件和渲染缓冲对象附件？

渲染缓冲对象能为你的帧缓冲对象提供一些优化，但知道什么时候使用渲染缓冲对象，什么时候使用纹理是很重要的。通常的规则是，如果你不需要从一个缓冲中采样数据，那么对这个缓冲使用渲染缓冲对象会是明智的选择。如果你需要从缓冲中采样颜色或深度值等数据，那么你应该选择纹理附件。性能方面它不会产生非常大的影响的。
